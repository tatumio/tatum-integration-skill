# Tatum Data API v4 - Endpoint Chain Support Mapping

## Overview

This directory contains a comprehensive mapping of **all Tatum Data API v4 endpoints** to their supported blockchain networks. The mapping was automatically generated by parsing the OpenAPI specification and extracting chain support information from each endpoint.

## Files

- **`endpoint-chain-support.json`** - Complete mapping of endpoints to supported chains
- **`ENDPOINT_CHAIN_MAPPING_README.md`** - This documentation file

## Data Structure

### Metadata
```json
{
  "metadata": {
    "totalEndpoints": 74,
    "totalCategories": 11,
    "totalUniqueChains": 78,
    "generatedAt": "2026-04-17",
    "source": "core-api/libs/api/data-api/src/lib/openapi.yaml"
  }
}
```

### Endpoints
Each endpoint contains the following information:

```json
{
  "GET /v4/data/collections": {
    "method": "GET",
    "supportedChains": ["ethereum-mainnet", "ethereum-sepolia", ...],
    "chainEnumSchema": "ChainEnumForCollections",
    "chainType": "EVM + Other",
    "category": "NFT API",
    "totalChains": 20,
    "summary": "Get tokens from a collection",
    "operationId": "GetCollectionsV4"
  }
}
```

**Fields:**
- `method` - HTTP method (GET, POST, PUT, DELETE)
- `supportedChains` - Array of supported blockchain identifiers
- `chainEnumSchema` - Reference to the ChainEnum schema in OpenAPI spec
- `chainType` - Categorization of chain support (see below)
- `category` - API category from OpenAPI tags
- `totalChains` - Number of supported chains
- `summary` - Brief description of the endpoint
- `operationId` - OpenAPI operation ID

### Chain Types

Endpoints are categorized by the type of chains they support:

1. **EVM-only** (34 endpoints)
   - Supports only EVM-compatible chains (Ethereum, BSC, Polygon, etc.)

2. **EVM + Other** (17 endpoints)
   - Supports EVM chains plus non-EVM chains like Solana, Tezos, etc.

3. **EVM + UTXO** (8 endpoints)
   - Supports both EVM and UTXO-based chains (Bitcoin, Litecoin, etc.)

4. **UTXO-only** (4 endpoints)
   - Supports only UTXO-based chains

5. **Multiple chains** (4 endpoints)
   - Supports various chain types that don't fit other categories

6. **Single chain** (6 endpoints)
   - Chain-agnostic endpoints (e.g., exchange rate by symbol)

7. **Solana-only** (1 endpoint)
   - Supports only Solana network

## Usage Examples

### Find endpoints supporting a specific chain

```javascript
// Example: Find all endpoints that support Polygon mainnet
const data = require('./endpoint-chain-support.json');

const polygonEndpoints = Object.entries(data.endpoints)
  .filter(([key, value]) => value.supportedChains.includes('polygon-mainnet'))
  .map(([key, value]) => ({
    endpoint: key,
    category: value.category,
    totalChains: value.totalChains
  }));

console.log(polygonEndpoints);
```

### Get all endpoints in a category

```javascript
// Example: Get all NFT API endpoints
const data = require('./endpoint-chain-support.json');

const nftEndpoints = data.categories['NFT API'];
console.log(nftEndpoints);
```

### Filter by chain type

```javascript
// Example: Get all UTXO-only endpoints
const data = require('./endpoint-chain-support.json');

const utxoEndpoints = data.chainTypes['UTXO-only'];
console.log(utxoEndpoints);
```

### Check if an endpoint supports a chain

```javascript
// Example: Check if an endpoint supports Solana
const data = require('./endpoint-chain-support.json');

const endpoint = 'GET /v4/data/wallet/portfolio';
const supportsChain = data.endpoints[endpoint].supportedChains.includes('solana-mainnet');

console.log(`${endpoint} supports Solana: ${supportsChain}`);
```

## Special Cases

### Chain-Agnostic Endpoints

Some endpoints don't require a `chain` parameter (6 endpoints with "Single chain" type):
- `/v4/data/rate/symbol` - Exchange rates by symbol (e.g., BTC, ETH)
- `/v4/data/rate/symbol/batch` - Batch exchange rates
- `/v4/data/rate/symbol/OHLCV` - Historical OHLCV data
- `/v4/data/rate/symbol/OHLCV/batch` - Batch OHLCV data
- `/v4/data/rate/price-change` - Price change data
- `/v4/data/rate/price-change/batch` - Batch price change

These endpoints work with currency symbols instead of blockchain identifiers.

### Inline Enum Definitions

Some endpoints define chain enums inline rather than referencing a shared schema:
- `/v4/data/multitoken/balances` - Has inline enum with 19 chains
- `/v4/data/staking/*` endpoints - Have inline enums with 1-2 chains

These are marked with `"chainEnumSchema": "inline-enum"` in the mapping.

### Batch Endpoints

POST endpoints with batch operations use array request bodies:
- The chain parameter is inside each item of the array
- Example: `POST /v4/data/rate/contract/batch`

## Maintenance

To regenerate this mapping after OpenAPI spec updates:

1. Ensure the OpenAPI spec is at: `core-api/libs/api/data-api/src/lib/openapi.yaml`
2. Run the parser script: `python3 parse_openapi.py`
3. The output will be written to: `endpoint-chain-support.json`

## Parser Implementation

The mapping is generated by a Python script that:
1. Loads the OpenAPI YAML specification
2. Iterates through all `/v4/data/*` paths
3. For each HTTP method (GET, POST, PUT, DELETE):
   - Extracts the `chain` parameter from query params or request body
   - Resolves the ChainEnum schema reference
   - Extracts the list of supported chains
   - Categorizes by chain type
   - Captures metadata (category, summary, operation ID)
4. Outputs a comprehensive JSON mapping

The parser handles:
- Schema references (`$ref`)
- Inline enum definitions
- Array request bodies with nested schemas
- Multiple HTTP methods per path
